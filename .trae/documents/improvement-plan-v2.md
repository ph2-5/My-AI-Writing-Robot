# AI写字机器人 - 完善计划 V2（基于市场调研）

## 市场调研结论

通过对奎享雕刻、广库、书写助手、手写精灵、快抄宝、墨迹书写等主流写字机器人软件的调研，发现以下关键功能差异：

### 竞品核心功能对比

| 功能 | 奎享雕刻 | 广库 | 书写助手 | 快抄宝 | 我们的系统 |
|------|---------|------|---------|--------|-----------|
| 纸张模板 | 横线/方格/空白/自定义 | 横线/方格/空白 | 横线/方格 | 横线/方格 | 仅空白A4 |
| 字体库 | 万种字体+TTF导入 | 系统字体+仿手写 | 系统字体 | 系统字体 | Hershey单线 |
| 字体变形 | 形状/笔锋/浮动/简化 | 随机变形 | 基础变形 | 无 | 抖动+拐角 |
| 字间距调节 | 有 | 有 | 有 | 有 | 固定1.2倍 |
| 行间距调节 | 有 | 有 | 有 | 有 | 有 |
| 笔画粗细 | 有（压力控制） | 有 | 无 | 无 | 无 |
| 笔迹模仿 | 支持上传笔迹 | 支持 | 不支持 | 不支持 | 不支持 |
| 多页排版 | 支持 | 支持 | 支持 | 支持 | 基础换页 |
| 自动翻页 | 部分型号支持 | 部分支持 | 不支持 | 不支持 | 不支持 |
| 批量处理 | 不支持 | 不支持 | 不支持 | 不支持 | 不支持 |
| AI智能分析 | 不支持 | 不支持 | 不支持 | 不支持 | 支持 |
| Word导入 | 不支持 | 不支持 | 不支持 | 不支持 | 支持 |
| UML图形 | 不支持 | 不支持 | 不支持 | 不支持 | 支持 |
| Web界面 | 无 | 无 | 无 | 无 | 有 |

### 关键发现

1. **我们的核心优势**：AI智能分析 + Word导入 + UML图形生成 + Web界面 — 这些是市面上所有竞品都没有的差异化功能
2. **我们的明显短板**：纸张模板单一、无TTF字体导入、无笔迹模仿、无笔画粗细控制、无批量处理
3. **竞品普遍功能**：纸张模板（横线/方格/空白）、字体变形（形状/笔锋/浮动）、字间距调节、多页排版

## 完善计划（16项 → 22项）

### 一、前端完善（新增4项）

#### 1. 参数面板增加滑块控制（P1）
**问题**：当前所有参数只有数字输入框，没有滑块，用户体验差。
**方案**：为每个参数添加 range 滑块（min/max 范围），滑块和数字输入框双向同步。
**涉及文件**：PaperSettings、FontSettings、RobotSettings、HandDrawnSettings

#### 2. 参数预设/重置功能（P1）
**问题**：没有一键恢复默认参数的功能，用户误改后无法快速恢复。
**方案**：添加「重置默认」按钮 + 「保存预设」/「加载预设」（localStorage）
**涉及文件**：useConfigStore.ts、Home.tsx

#### 3. SVG 预览增强——缩放平移（P2）
**问题**：SVG 预览区域没有缩放、平移功能，大图纸看不清细节。
**方案**：添加缩放控制（+/-/100% + 鼠标滚轮）+ 拖拽平移 + 显示/隐藏纸张边框和可写区域
**涉及文件**：SvgPreview.tsx

#### 4. 文件上传增强（P2）
**问题**：上传后没有显示文件信息，没有删除/重新上传功能。
**方案**：显示文件名和大小 + 「删除」按钮 + 支持多文件上传队列
**涉及文件**：FileUpload.tsx

#### 5. 生成结果历史记录（P2）
**问题**：每次生成新结果会覆盖旧结果，无法对比。
**方案**：添加「历史记录」列表，显示时间/文件名/笔画数，点击可切换预览
**涉及文件**：useGenerateStore.ts、新建 HistoryList.tsx

#### 6. 响应式布局（P3）
**问题**：当前布局是固定 320px 左侧面板，小屏幕体验差。
**方案**：移动端适配（< 1024px 时左侧面板可折叠）+ 面板宽度可拖拽调整
**涉及文件**：Home.tsx

### 二、后端完善（新增1项）

#### 7. 配置热重载机制（P0）
**问题**：`load_config_from_dict` 直接修改全局模块变量，多线程/并发请求时产生竞态条件。
**方案**：将 `config.py` 改为 Config 类实例，各模块接收 Config 参数
**涉及文件**：config.py、main.py、layout_engine.py、hand_drawn.py、command_generator.py

#### 8. 并发安全与错误处理（P0）
**问题**：没有超时控制、请求队列、友好错误返回。
**方案**：`runPython` 添加 5 分钟超时 + 请求锁 + 统一错误响应
**涉及文件**：api/routes/homework.ts

#### 9. 临时文件清理（P2）
**问题**：uploads 和 outputs 目录文件无限累积。
**方案**：启动时清理超过 24 小时的旧文件
**涉及文件**：api/routes/homework.ts、api/server.ts

### 三、核心功能完善（新增6项，基于市场调研）

#### 10. LLM 分析支持流式输出（P1）
**问题**：LLM 分析同步阻塞，用户看不到进度。
**方案**：SSE 流式返回 LLM 分析进度，前端显示进度条
**涉及文件**：api/routes/homework.ts、homework_analyzer.py、GeneratePanel.tsx

#### 11. 纸张模板系统（P1）【新增】
**市场调研**：奎享雕刻、广库、书写助手都支持横线本/方格本/空白本/自定义模板。
**问题**：当前只支持空白A4纸，不支持横线本、方格本、笔记本等常见格式。
**方案**：
- 添加纸张模板选择器：空白A4 / 横线本 / 方格本 / 笔记本 / 自定义
- 横线本：可设置行数、行高、页边距
- 方格本：可设置格子大小、行列数
- 笔记本：可设置装订线位置、打孔位置
- 自定义：用户输入纸张尺寸、行高、格子大小
- 排版引擎根据模板类型调整绘图逻辑（文字对齐到行/格子）
**涉及文件**：
- `src/components/PaperTemplateSelector.tsx`（新增）
- `layout/layout_engine.py` — 添加模板对齐逻辑
- `layout/models.py` — 添加 PaperTemplate 模型
- `output/command_generator.py` — 模板背景线输出（虚线/点线）

#### 12. TTF 字体导入与笔迹模仿（P1）【新增】
**市场调研**：奎享雕刻支持万种字体+TTF导入，广库支持仿手写和笔迹模仿，HandtextAI支持个人字体生成。
**问题**：当前只有内置 Hershey 字体（50个汉字），无法使用用户自己的字体，无法模仿真人笔迹。
**方案**：
- 前端支持上传 TTF/OTF 字体文件
- 后端使用 `fontTools` 提取字体轮廓
- 轮廓中心线化算法（将填充字体转为单线笔画）
- 笔迹模仿：用户上传手写样本图片，AI 提取笔画特征生成个性化字体
- 字体回退机制：找不到字形时尝试近似匹配或显示占位符
**涉及文件**：
- `src/components/FontUpload.tsx`（新增）
- `fonts/ttf_converter.py`（新增）— fontTools 轮廓提取+中心线化
- `fonts/hershey.py` — 添加字体回退机制
- `api/routes/homework.ts` — 字体上传接口

#### 13. 字体变形增强（P2）【新增】
**市场调研**：奎享雕刻支持形状变形、笔画笔锋、上下浮动、字体简化；竞品普遍支持字间距、笔画粗细调节。
**问题**：当前只有基础抖动（0.4mm振幅）和拐角增强，缺少：字间距随机变化、笔画粗细模拟、字形微变形、上下浮动。
**方案**：
- 字间距随机变化：每个字符间距 × random(0.85, 1.15)
- 笔画粗细模拟：通过速度变化模拟（慢速=粗，快速=细）
- 字形微变形：整体倾斜 ±2°、字符高度随机压缩/拉伸 ±5%
- 上下浮动：整行 baseline 随机偏移 ±0.5mm
- 笔锋模拟：笔画起止点添加「顿笔」效果（小圆点）
**涉及文件**：
- `effects/hand_drawn.py` — 扩展变形算法
- `src/components/HandDrawnSettings.tsx` — 添加新参数

#### 14. 字间距独立控制（P2）【新增】
**市场调研**：所有竞品都支持字间距独立调节。
**问题**：当前字间距固定为 font_size × 1.2，无法调节。
**方案**：添加 `charSpacing` 参数（0.5-2.0 倍，默认 1.2）
**涉及文件**：
- `config.py` — 添加 CHAR_SPACING
- `fonts/hershey.py` — `get_char_width` 使用动态间距
- `src/components/FontSettings.tsx` — 添加字间距输入

#### 15. 多页排版与分页预览（P2）
**问题**：换页逻辑简单，没有真正的分页排版和翻页预览。
**方案**：`LayoutResult` 添加 `pages` 结构，每页独立 SVG viewBox，前端支持翻页
**涉及文件**：layout/models.py、layout_engine.py、command_generator.py、SvgPreview.tsx

#### 16. 题目类型扩展（P2）
**问题**：只支持 UML 用例图/类图/序列图/活动图和文字题。
**方案**：添加状态图、组件图、部署图、代码题、表格题
**涉及文件**：parsers/homework_parser.py、layout/layout_engine.py、analyzers/homework_analyzer.py

#### 17. 批量处理与任务队列（P2）【新增】
**市场调研**：目前所有竞品都不支持批量处理，这是一个差异化机会。
**问题**：每次只能处理一个 Word 文档，无法批量处理多个作业。
**方案**：
- 支持多文件上传队列
- 后台任务队列（异步处理）
- 任务状态实时推送（SSE）
- 批量下载（ZIP 打包）
**涉及文件**：
- `src/components/BatchQueue.tsx`（新增）
- `api/routes/homework.ts` — 添加批量上传/状态查询接口
- `api/queue.ts`（新增）— 任务队列管理

- 纸张尺寸改变时，自动调整边距比例
- 提供「快速设置」预设：小字密排 / 大字疏排 / 标准作业
**涉及文件**：

## 关键决策

1. **TTF 字体导入 vs Hershey 扩展**：优先 TTF 导入（fontTools），因为 Hershey 中文字库维护成本高，TTF 导入可以立即支持所有字体
2. **纸张模板优先级**：横线本 > 方格本 > 笔记本，横线本是最常用的作业本格式
3. **批量处理价值**：虽然竞品都没有，但这是明显的差异化功能，值得投入
